A BLIT386 demo is just a plain class with up to four lifecycle methods, nothing more. You hand the class to `bootstrap`,
and the engine takes care of the boring parts – canvas setup, the game loop, picking a backend. You write the four
methods, it runs them.

## The four lifecycle methods

### configure (optional)

`configure` runs before anything else. Return a partial `HardwareSettings` object to override the canvas resolution,
target frame rate, or rendering backend. Omit any field to keep the default.

```ts twoslash
import { bootstrap, BT, Vector2i } from 'blit386';
import type { HardwareSettings } from 'blit386';

class MyDemo {
  configure(): Partial<HardwareSettings> {
    return { displaySize: new Vector2i(160, 120), targetFPS: 30 };
  }
}

bootstrap(MyDemo);
```

Omit `configure` entirely and the engine uses 320x240 at 60 FPS with WebGPU.

### init (async)

`init` runs once after the canvas is ready. Load palettes, sprite sheets, and fonts here. The engine waits for the
returned promise to settle before starting the loop, so every `await` is safe. Return `true` when setup succeeds, or
`false` to abort startup.

```ts twoslash
import { BT, Palette } from 'blit386';

class MyDemo {
  // ---cut-before---
  async init() {
    BT.paletteSet(Palette.vga());
    return true;
  }
  // ---cut-after---
}
```

### update

`update` runs on every frame before rendering. Advance game state, read input, and move objects here. Do not issue draw
calls from `update` – those belong in `render`.

```ts twoslash
class MyDemo {
  private x = 0;
  // ---cut-before---
  update() {
    this.x = (this.x + 2) % 320;
  }
  // ---cut-after---
}
```

### render

`render` runs after `update` on every frame. Clear the screen and issue all draw calls here.

```ts twoslash
import { BT, Rect2i } from 'blit386';

class MyDemo {
  private x = 0;
  // ---cut-before---
  render() {
    BT.clear(0);
    BT.drawRectFill(new Rect2i(this.x, 112, 16, 16), 15);
  }
  // ---cut-after---
}
```

## A complete example

```ts twoslash
import { bootstrap, BT, Palette, Rect2i, Vector2i } from 'blit386';
import type { HardwareSettings } from 'blit386';

class SliderDemo {
  private x = 0;

  configure(): Partial<HardwareSettings> {
    return { displaySize: new Vector2i(320, 240), targetFPS: 60 };
  }

  async init() {
    BT.paletteSet(Palette.vga());
    return true;
  }

  update() {
    this.x = (this.x + 2) % 320;
  }

  render() {
    BT.clear(0);
    BT.drawRectFill(new Rect2i(this.x, 112, 16, 16), 15);
  }
}

bootstrap(SliderDemo);
```

Pass the class itself to `bootstrap` – not an instance. The engine constructs it, calls `configure`, sets up the canvas,
resolves `init`, then loops `update` and `render` at the target frame rate.

## Going further

There are over 30 self-contained examples over at [demos.blit386.dev](https://demos.blit386.dev) – sprites, palette
cycling, pointer and keyboard input, a Snake game. Each one is a single JavaScript file, so you can read the whole thing
in one sitting, right next to these docs. That is honestly the fastest way to learn the loop: find a demo that does
something close to what you want, and read it.
