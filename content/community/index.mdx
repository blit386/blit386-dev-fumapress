BLIT386 is a small, focused project. The best way to get involved is to pick the channel that matches your intent.

## Ask questions and discuss

Have a question about the API, a demo you are building, or how something works under the hood? Start a discussion or
open an issue on GitHub — both are good places for back-and-forth.

- **[GitHub Discussions](https://github.com/blit386/blit386/discussions)** — long-form questions, ideas, and
  show-and-tell
- **[Discord](https://discord.gg/tC2wGt88Uj)** — real-time chat, quick questions, sharing work in progress

## Report bugs

Found something broken? Open an issue on the main engine repository:

- [github.com/blit386/blit386/issues](https://github.com/blit386/blit386/issues) — engine bugs, API feedback, feature
  requests

Please include a minimal reproduction and the browser and GPU you are using. WebGPU behavior can vary across drivers, so
details matter.

## Follow updates

Releases, new demos, and project news are posted through these channels:

- **[X / Twitter](https://x.com/blit386)**
- **Bluesky** (coming soon)
- **Mastodon** (coming soon)
- **GitHub releases** — [github.com/blit386/blit386/releases](https://github.com/blit386/blit386/releases)

The GitHub releases page is the most reliable source — every version bump gets a release note with a changelog.
